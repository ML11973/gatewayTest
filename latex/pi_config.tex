\hypertarget{pi_config_autotoc_md5}{}\doxysection{Buildroot}\label{pi_config_autotoc_md5}
Buildroot is a toolchain that simplifies custom Linux target creation. It is available on the \href{https://buildroot.org/}{\texttt{ Buildroot project website}}. It is an archive to be downloaded and compiled on another host system. It also provides cross-\/compilers. These are useful to quickly compile the Linux kernel and whatever program to be developped using the host machine\textquotesingle{}s considerably higher computing power.

The reason Buildroot is interesting is that it allows easy configuration of a custom Linux system. We can tinker with the kernel and configurations in order to only get the functionalities we need. This allows faster boot times and security upgrades.\hypertarget{pi_config_autotoc_md6}{}\doxysubsection{Buildroot configuration and initial build}\label{pi_config_autotoc_md6}

The device tree overlay is a file that describes the hardware setup so that Linux can know where its peripherals are. It doesn\textquotesingle{}t describe all hardware by default, to keep a lightweight system.

Since the gateway application uses the CM4\textquotesingle{}s serial device 4, we have to tell Linux it exists and how. Fortunately, the tedious description work has already been done for us. All we need to do is add the following line at the end of /boot/ dtoverlay.


\begin{DoxyCode}{0}
\DoxyCodeLine{\# enable UART4}
\DoxyCodeLine{dtoverlay=uart4}

\end{DoxyCode}
