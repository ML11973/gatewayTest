\hypertarget{ISM3_manual_autotoc_md68}{}\doxysubsection{Overview}\label{ISM3_manual_autotoc_md68}
The ISM3 radio module is a serial-\/accessible radio module for LPWAN applications. It operates in the ISM 868 or 915 MHz band.

Note\+: ISM stands for \href{https://en.wikipedia.org/wiki/ISM_radio_band}{\texttt{ Industrial-\/\+Scientific-\/\+Medical}}, which refers to the common name given to its operating band. The ISM 868 band is an unlicensed (meaning anyone can use it as long as they follow some common rules) band with center frequency at 868 MHz. It is used for IoT applications in Europe. Its US counterpart is at 915 MHz. There are other ISM bands, such as the 2.\+4 and 5 GHz bands used by Wi-\/\+Fi.\hypertarget{ISM3_manual_autotoc_md69}{}\doxysubsection{Usage}\label{ISM3_manual_autotoc_md69}
Initialize handler functions (\mbox{\hyperlink{ism3_8h_a57d2806f433f049b29779b8a9040f460}{ism\+\_\+init}} with \mbox{\hyperlink{ism3__handlers_8h}{ism3\+\_\+handlers.\+h}} as arguments), configure module and set sync mode.

\mbox{\hyperlink{ism3_8h_a11fadec027d13495865f2e9e9853fd1e}{ism\+\_\+tick}} regularly to update serial RX/\+TX buffers. Without ticking, data may neither be transmitted nor received.

Transmit frames with \mbox{\hyperlink{ism3_8h_aa01f3c3a9529ea62bc6140984885c7a7}{ism\+\_\+tx}}.

\hypertarget{ISM3_config_autotoc_md80}{}\doxyparagraph{BAUDRATE macro}\label{ISM3_config_autotoc_md80}
This macro defines linux hardware baudrate. It should match baudrate set in configuration\+\_\+commands. Default value is 19200. It should not be set as a random value since CM4 hardware only supports a few fixed baudrates. See linux\+\_\+uart\+\_\+init source in \mbox{\hyperlink{buffered__uart_8c}{buffered\+\_\+uart.\+c}} and \href{https://blog.mbedded.ninja/programming/operating-systems/linux/linux-serial-ports-using-c-cpp/\#baud-rate}{\texttt{ this blog post}} for additional information about UNIX compliant baudrates.\hypertarget{ISM3_config_autotoc_md81}{}\doxysubsubsection{What can I change?}\label{ISM3_config_autotoc_md81}
You can change host baudrate, pattern, TX retry count, GPIO signals, Sync RX, RF Phy and channels, data slot count, sync mode, sync rx interval, power, address and group.

To keep gateway functionality, do not change sync mode, address and group. \hypertarget{ISM3_syncmodes}{}\doxysection{ISM3 sync modes}\label{ISM3_syncmodes}
The ISM module used in has several sync modes for the user to choose from. The following descriptions should help the user choose the relevant mode for their application.\hypertarget{ISM3_syncmodes_autotoc_md82}{}\doxyparagraph{ISM\+\_\+\+TX}\label{ISM3_syncmodes_autotoc_md82}
Module is in gateway mode. It will be the master of its WPAN (Wireless Personal Area Network).\hypertarget{ISM3_syncmodes_autotoc_md83}{}\doxyparagraph{SM\+\_\+\+RX\+\_\+\+ACTIVE}\label{ISM3_syncmodes_autotoc_md83}
Module is in RX mode. It will remain in SYNCHRONIZED state all the time, and receive unicast and multicast frames. The module will not enter power saving mode.\hypertarget{ISM3_syncmodes_autotoc_md84}{}\doxyparagraph{SM\+\_\+\+RX\+\_\+\+LOW\+\_\+\+POWER}\label{ISM3_syncmodes_autotoc_md84}
Module is in low power mode. It will not react to group wakeups and remain asleep. The only way for the module to receive data is to get it from a beacon data change.\hypertarget{ISM3_syncmodes_autotoc_md85}{}\doxyparagraph{SM\+\_\+\+RX\+\_\+\+LOW\+\_\+\+POWER\+\_\+\+GROUP}\label{ISM3_syncmodes_autotoc_md85}
Module is in low power mode and will react to group wakeups. When the gateway wakes one of the groups the module belongs to, it will enter state ISM\+\_\+\+SYNCHRONIZED. When the node is synchronized, it will receive unicast and multicast frames. \hypertarget{ISM3_states}{}\doxysection{ISM3 states}\label{ISM3_states}
General information\+: It takes approximately 15 seconds for a node to lose sync with the current configuration in \mbox{\hyperlink{ism3_8c}{ism3.\+c}}. This is because the node tries several times to reach the gateway until it determines it has lost sync.\hypertarget{ISM3_states_autotoc_md86}{}\doxyparagraph{ISM\+\_\+\+OFF}\label{ISM3_states_autotoc_md86}
Module is uninitialized.\hypertarget{ISM3_states_autotoc_md87}{}\doxyparagraph{ISM\+\_\+\+NOT\+\_\+\+SYNCHRONIZED}\label{ISM3_states_autotoc_md87}
Module is waiting for sync. It is not connected to a gateway.\hypertarget{ISM3_states_autotoc_md88}{}\doxyparagraph{ISM\+\_\+\+SYNCHRONIZED}\label{ISM3_states_autotoc_md88}
Module is synchronized to a gateway. Gets in this state on wakeup from gateway if it is in SM\+\_\+\+RX\+\_\+\+LOW\+\_\+\+POWER\+\_\+\+GROUP power mode. Stays in this state for as long as it is connected in SM\+\_\+\+RX\+\_\+\+ACTIVE power mode.\hypertarget{ISM3_states_autotoc_md89}{}\doxyparagraph{ISM\+\_\+\+LOW\+\_\+\+POWER\+\_\+\+SYNC}\label{ISM3_states_autotoc_md89}
Module is synced to a gateway in SM\+\_\+\+LOW\+\_\+\+POWER or SM\+\_\+\+LOW\+\_\+\+POWER\+\_\+\+GROUP power mode, but not woken up. Cannot be woken up in SM\+\_\+\+LOW\+\_\+\+POWER.\hypertarget{ISM3_states_autotoc_md90}{}\doxyparagraph{ISM\+\_\+\+TX\+\_\+\+SYNC}\label{ISM3_states_autotoc_md90}
Module is the gateway of its WPAN.\hypertarget{ISM3_states_autotoc_md91}{}\doxyparagraph{ISM\+\_\+\+VERSION\+\_\+\+READY}\label{ISM3_states_autotoc_md91}
Module is getting initialized and has not received a power mode. \hypertarget{ISM3_troubleshooting}{}\doxysection{ISM3 Troubleshooting}\label{ISM3_troubleshooting}

\hypertarget{ISM3_troubleshooting_autotoc_md100}{}\doxysubsubsection{Radio communication}\label{ISM3_troubleshooting_autotoc_md100}
If configuration LEDs light up, radio link may be defective. Check R15-\/\+R16 position matches desired antenna setup. Check impedance matching on C21, C23, C24. If using on-\/shield antenna, check C25 and C26.
